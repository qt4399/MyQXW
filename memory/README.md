`memory/` 目录负责保存 Agent 的规则、短期记忆、日记忆、月归档和运行状态。

## 目录结构

### `memory/md/`

用于存放自然语言文档和长期可读记忆：

- `AGENT.md`：系统信息与记忆原则
- `ROLE.md`：身份设定
- `RELATION.md`：关系设定
- `SOUL.md`：灵魂/人格核心设定
- `HEARTBEATS.md`：心跳协议
- `day.md`：当前记忆日内容
- `month.md`：最近 30 天归档内容

### `memory/yaml/`

用于存放结构化数据：

- `state.yaml`：与记忆相关的持久状态、时间戳和 PLAY 配置
- `communicate.yaml`：最近 20 轮对话窗口
- `temp_communicate.yaml`：等待心跳整理的溢出对话

## 各文件职责

### `state.yaml`

负责保存当前运行状态，例如：

- 当前记忆日
- 最近一次用户消息时间
- 最近一次助手消息时间
- 最近一次心跳时间
- 最近一次日归档时间
- 最近一次临时对话整理提醒时间
- 最近一次临时对话整理完成时间
- `play` 的启用/激活状态与触发时间

运行中的聊天/心跳服务由 `logic/chat_service.py` 和 `congnition/heart_service.py` 在内存里运行，不再把 `chat / idle / heartbeat` 这类临时模式落到 `state.yaml`。

其中：

- `last_temp_digest_prompted_at`：上一次系统提醒整理临时对话的时间，用于提醒冷却。
- `last_temp_digest_completed_at`：上一次真正删除已处理临时轮次的时间，用于记录整理完成。

### `communicate.yaml`

- 保存最近 20 轮对话
- 普通对话时直接注入到 prompt 中
- 超过 20 轮后，最旧轮次会被移到 `temp_communicate.yaml`

### `temp_communicate.yaml`

- 保存已经溢出的旧对话
- 不直接注入普通对话
- 由心跳中的 Agent 决定是否整理进 `day.md`
- 整理完成后，对应轮次应被删除

### `day.md`

`day.md` 是当前记忆日的工作台，由 Agent 在心跳里不断整理。

推荐结构：

```md
# day.md - 当日记录

日期: 2026-03-11

## 概括
- 今天大概发生了什么

## 详细
- [时间] 更具体的事件
```

职责是：

- 保存当前记忆日的重要事件
- 保存当前记忆日的主题整理结果
- 保存需要在当天继续参考的细节

### `month.md`

`month.md` 用来保存最近 30 天的归档结果。

推荐结构：

```md
# month.md - 最近30天记录

## 2026-03-11
### 概括
- 这一天大概发生了什么

### 详细
- 这一天更详细的整理内容
```

职责是：

- 保存最近 30 天的每日归档
- 为普通对话提供“最近 30 天概括”注入内容
- 为按日期检索某一天完整内容提供来源

## 时间边界

memory 系统使用“记忆日”概念，而不是自然日 00:00 立即切换。

规则如下：

- 凌晨 4:00 之前，仍属于前一个记忆日
- 凌晨 4:00 之后，旧的 `day.md` 会归档进 `month.md`
- 然后系统开始新的 `day.md`

## 数据流

### 普通对话数据流

1. 用户提问
2. 注入最近 6 轮对话 + `day.md` + 最近 30 天概括
3. 模型回复
4. 这一轮对话写入 `communicate.yaml`
5. 如果超出 20 轮，则最老对话溢出到 `temp_communicate.yaml`

### 心跳整理数据流

1. 统一调度器里按自适应节流触发 heartbeat；调试模式下 `heart.py` 才会每秒发送一次 `Boom`
2. 心跳前更新状态
3. 检查是否跨过凌晨 4:00
4. 如果跨过，则归档 `day.md` 到 `month.md`
5. 检查 `temp_communicate.yaml` 是否值得整理
6. 如果值得，Agent 可将有用事件和主题写入 `day.md`
7. 已处理的临时轮次可从 `temp_communicate.yaml` 删除

## 读取策略

### 默认注入

系统默认注入：

- `day.md`
- 最近 30 天概括
- 最近 6 轮对话

### 按需读取

如果 Agent 想查看本月某一天的完整细节：

1. 先看系统已注入的“最近 30 天概括”
2. 找到一个明确日期，例如 `2026-03-10`
3. 调用 `read_month_day`
4. 读取该日期在 `month.md` 中的完整内容

注意：

- 不要把“前几天”“上周某天”直接传给工具
- 工具参数必须是 `YYYY-MM-DD`
