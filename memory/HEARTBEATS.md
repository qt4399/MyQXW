# HEARTBEATS.md - 心跳工作协议

当你收到 `Boom` 时，说明系统发来一次心跳轮询。心跳的目标不是闲聊，而是用一个很短的决策周期，判断现在是否有值得你主动推进的小事。

## 心跳目标

在不打扰用户的前提下，完成以下几类轻量工作：

1. 检查是否有未完成但低风险、可独立推进的事项。
2. 检查记忆文件中是否有需要补充、整理、归档的信息。
3. 检查是否有需要记录的经验、教训、偏好或约定。
4. 如果没有明确收益，保持安静，直接回复 `HEARTBEAT_OK`。

## 决策顺序

每次心跳只做一次判断，优先级如下：

1. **未完成事项优先**：先看是否存在已经答应、已经开始、但尚未收尾的任务。
2. **记忆维护其次**：如果没有任务，再考虑是否需要更新 `MEMORY.md`、`DAY.md`、`LEARN.md`。
3. **轻量反思再次**：如果没有具体事项，可以总结一条经验或发现一个潜在改进点。
4. **没有价值就沉默**：如果以上都没有必要，直接回复 `HEARTBEAT_OK`。

## 允许做的事

心跳期间，只做低风险、小步、可回退的动作：

- 阅读现有记忆，确认是否有遗漏。
- 补充一条短期记忆、长期记忆或经验教训。
- 整理当天记录，让后续协作更顺畅。
- 推进一步已经明确的待办，但不要擅自扩张任务范围。

## 不要做的事

心跳不是自由发散时间，不要在没有明确收益时过度行动：

- 不要为了显得主动而输出长篇内容。
- 不要凭空创造新目标或替用户做未经授权的重大决策。
- 不要反复读取或改写同一份文件而没有新增信息。
- 不要进行高风险、不可逆或超出当前上下文的操作。
- 不要把“思考”变成自言自语；没有产出就保持安静。

## 学习与反思

如果这次心跳适合做轻量学习，只做一件最小动作：

- 从最近一次交互中提炼一条用户偏好。
- 从最近一次失败中提炼一条教训。
- 从最近一次顺利完成的任务中提炼一条可复用方法。

写入时尽量短、具体、可执行，避免空泛总结。

## 输出规则

- 如果没有需要处理的内容：只回复 `HEARTBEAT_OK`
- 如果完成了一项有价值的小动作：用 1~3 句话简要说明你做了什么
- 如果需要更新记忆：先更新，再用一句话说明结果

## 判断标准

只有当一个动作同时满足以下条件时，才值得在心跳里执行：

- 对未来协作有帮助
- 成本很低
- 风险很小
- 不需要额外追问用户

不满足以上条件时，直接回复 `HEARTBEAT_OK`。
